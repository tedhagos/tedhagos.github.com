JavaScript is a loose, dynamically typed, interpreted, object oriented and functional programming language. None of that probably means anything right now, so why did I even bring it up. Because it sounds cool and it does give a bit of flare to the language. Makes it sounds really modern. The truth is it has around for quite some time (1995). It began its life under a different name (Mocha, then became LiveScript) when it was under the care of Netscape Communications. It was later renamed to JavaScript around the time when plugin support for Java was built into Netscape Navigator 1.

Java and JavaScript has got nothing to do with each other. They are two very different languages/technologies. They simply both java Java in their names. The similarities end there.

The popularity of JavaScript could be attributed to it's ubiquity. It's humble beginnings was simply aimed at providing some ammunication to web designers so they can "add a bit of smarts to the web pages" 2, Brendan Eich (creator of JavaScript) said so during one of his interviews. But now, JavaScript is no longer confined to web pages. You will also find it on smart phones, tablets, game consoles, desktops and server side applications.

4.1 TODO ECMA and JavaScript

When JavaScript, and web pages, were gaining a lot of popularity, the browser wars began. Netscape Communiations had, of course, Netscape Navigator and Microsoft had IE. There were other players then but these two were the main actors. Microsoft could not call their scripting languages "JavaScript" because of licensing issues so they called it JScript. For the most part the two languages were compatible, but there are differences still. This is where a standards organization can really be useful.

A standards body can establish a corpus of rules as to what makes up a browser side scripting language. What kind of capabilities should it have
