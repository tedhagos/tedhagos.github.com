Upgrading to OSX Yosemite

Which hardware can upgrade

If you can run Mavericks, you should be able to run Yosemite. If you are running Mavericks right now, the upgrade to Yosemite is a must, it fixes lots of things broken in Mavericks. Its snappier.

The upgrade process

  1. Make sure you have an updated Time Machine backup. If things go South, you have a way to go back
  2. Get the free upgrade from the App Store. Or get it from somebody who already downloaded the installer. The file size of the installer is above 5GB.
  3. Just follow the prompts of the upgrade process, its very straightforward
  4. Wait. This is the annoying part. The upgrade time of Yosemite is the longest in recent memory. I don't remember any upgrade taking this long. It might get stuck in "2 minutes remaining" for hours on end. Leave it be. Its best to do the upgrade overnight, on a weekend, when you don't have to use the machine for any work related stuff or anything urgent

Post upgrade

Go to the App Store update immediately after the upgrade has finished. Don't use anything else yet. You may need to get some updates for your other software so they can play nicely with Yosemite. Try to see if there are any available updates, install them immediately.

Things that might break

  1. You may have noticed that Dropbox is not running on task bar. If it is not, open the Activity Monitor (Go to Finder, click Applications, go to the Utilities folder then click Activity Monitor.app), do you see any "Dropbox not responding", if you do, double click on it. A dialog box will popup. Choose "Quit" or "Force Quit". Press cmd+spacebar (that will launch Spotlight), type "Dropbox", press the enter key
  2. If you are running Microsoft's OneDrive, you may have noticed that it is also not running. Just go to Spotlight (cmd+spacebar), type "OneDrive" and press enter.

Homebrew was broken

Homebrew uses Ruby 1.8 and Yosemite uses Ruby 2.0. This is why Homebrew doesn't work with 10.10. This would have been easily fixed if you did a brew update and brew upgrade before going up to Yosemite. Just in case you did upgrade to Yosemite without upgrading Homebrew, the fix is this

Edit the file /usr/local/Library/brew.rb

The first line on the file is 

#!/System/Library/Frameworks/Ruby.framework/Versions/1.8/usr/bin/ruby -W0

Replace the hard coded reference to the Ruby version number. The edited line should read 

#!/System/Library/Frameworks/Ruby.framework/Versions/Current/usr/bin/ruby -W0

You have to commit the changes you made on brew.rb before you can brew upgrade, otherwise, you will be back to square one. 

cd $(brew --repository)
git commit -a -m "changed brew rb to point to Ruby 2
brew update
brew upgrade

If the cd $(brew –repository) command gives you problem, just cd to usr/local/Library then do the commit while in that directory.

This fixed was written by Hans Van Gent. The published web page can be found at http://jcvangent.com/fixing-homebrew-os-x-10-10-yosemite/
