3 TODO Android

There is a much nicer version of this on thelogbox PhoneGap setup

  1. Download the SDK from developer.android.com/sdk. There are two options. One of the download options is an ADT for Eclipse — obviously, that is if you use the Eclipse IDE. The other one is just the plain SDK. The download page names this link "Use an Existing IDE". Get the just the plain SDK.
  2. Unzip or untar the downloaded installer
  3. Update the SYSTEM PATH to include ANDROID_HOME/tools and ANDROID_HOME/platform-tools, where ANDROID_HOME is the top level directory of what you just extracted in the previous step. When you extract the zipped or tar file, it will spit out a couple of files and executables. They are all contained on a folder. So when you extract them, they won't be all over the place. Usually the name of this folder is android-sdk-linux, if you are using osx, it will be android-sdk-osx and android-sdk-windows if you downloaded the Windows version
  4. The files you just downloaded aren't all of it. You are not nearly done yet. Those set of files contains just about enough so you can get to the internet pull a lot more files. Get a terminal window and type android, if you encounter an error messager, like "bad command or file name", go back to the previous step, make sure you updated the SYSTEM PATH properly to include the ANDROID_HOME/tools folder. That folder is where the android executable is located.
  5. You should be seeing a GUI dialog window now. It lists a lot of things that you've got to download. You don't need to download everything, just get what you need

The android SDK manager is a long list of tick boxes. It tells what tools and platforms you can download. It also tells you which ones are installed on your system and which ones are not.

4 Tools section

On the tools section, check the boxes Android SDK Tools, Android SDK Platform-tools and Android SDK Build-tools–just get the latest version of the build tools. It's easy to tell because they are numbered anyway, you can see which one is the latest version. The higher the number, the more recent it is.

5 Platform version (API levels)

Next is a long list of platform version. If you are more familiar with the Android dessert names like KitKat, Jelly Bean, Froyo etc, you won't see them here. Instead you will see API levels. API level 19 is KitKat, levels 16-18 is Jelly Bean. Ice Cream Sandwich is level 15. The Android Developer Site maintains a page of platform versions. You will the codenames (dessert names) with their corresponding API levels.

Check any API level that you think you will use. At a minimum you should get the SDK platform, and a couple of system images for each API level. The system images might be needed if you will use an Android Virtual Device for testing and debugging. On the other hand, if you will use a physical device for development and testing, then you can opt not to download the system images.

6 Extras

If you are on Windows, you should get the Google USB Driver. You will need this if you will use a physical Android device for development. For the rest, you can just come back for them when you need it.

7 Install the packages

Click the "Install packages" button. Accept the licenses. This will take time, everything you selected will be pulled down via the internet and will then be later reconfigured. All of the downloaded files will be placed inside the ANDROID_HOME folder
