2 TODO Development Environment

The chapter walks through some detailed steps for setting up an Android development environment. From downloading the necessary files to installing, configuring and making the necessary changes to OS system path. Android depends on a lot of things to be properly configured before you can do anything meaningful in it. At the time of writing, it requires any of the following Java SDKs — 1.5, 1.6 or 1.8. Although Java 8 is already out, the examples in the book were all tested using JDK 1.7.

The ADK (Android Development Kit) also needs a build tool. Either Apache Ant or Gradle can be used. The examples in the book used Apache Ant. Ant is a workhorse. It is a bit dated but tried and true.

5 DONE Theoretical Minimum

The chapter provides a quick introduction to the Java platform. It does not cover the entirety of the Java language. It does not intend to. A full treatment of the Java language fills a couple of books. A single, albeit, quite lengthy chapter was allocated to address the theoretical minimum only to serve as a basic grounding for a beginning programmer.

The chapter covers basic Java concepts such as declaration and access control, variables, types, control structures, objects, classes, database programming, XML and Ant

6 TODO Getting Started

Beginning programmers often need some quick wins and successes whenever they venture into uncharted territories. This chapter provides that. We dip our toes into the water, so to speak. It walks you through the process of building the most simple Android app. From creating the project using just the Android SDK, to compiling and running the application on a device. Its goal is provide a feel on the basic workflow, tasks and activities that are involved in the creation of an Android app.

7 TODO Activity and the UI

Mobile applications are pretty close to the user. It is high on utility and interaction. Most of the apps written for the mobile will have user interfaces. This chapters explores the ways on how to build UIs in Android. Both declaratively and programmatically.

9 TODO Debugging

Building an application is a difficult undertaking. There are a lot of things that will not go your way. When an application is not behaving as expected, you need to have to ability to watch as things happen inside your code. This chapter introduces the reader to debugging techniques in Android.

10 TODO Event handling

User interface involves widgets such as buttons, text input fields etc. These widgets are event aware. The user will do something with them. This chapter contains instructions on how to listen for those events so the programmer can write codes when something happens on the UI.

11 TODO Database programming

Some applications may not need to persist data. But those kinds of app will be few. A major class of mobile applications will need a way to store data. You can store data as preferences or in a file. This chapter deals with data storage using SQLite — the detault database of Android applications.

12 TODO Working with the network

More and more applications depend on network connectivity. Be it via wifi or cellular data. This chapter explores the basic steps on detecting connectivity.
