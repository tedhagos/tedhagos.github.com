Android applications are built using a number of ways and technologies. Some people build them using a language other than Java. There are ways to build Android Apps using JavaScript, CSS, HTML and Phone for example. Some use C# or Mirah (A Ruby-like language) to build Android apps. But this book is about building native Android applications. That means we will focus on the Java language as a means to build native Android applications. The theoretical minimum to do such a thing includes the following.

  1. Working knowledge of the Java language, specifically Java Standard Edition
  2. Understanding of XML. Its structure and uses
  3. Understanding of build tools such as Apache Ant or Gradle. In this book, we will use Apache Ant

This chapter is aimed to provide sufficient grounding and background on the three areas above. If you have programmed in Java before, this chapter might be a refresher. If you donâ€™t have Java programming, it could be an introduction.

This introductory material on Java is not intended to be a full treatment of the language and its associated technologies. The intent is arm the reader with enough conceptual and practical footing in order for the Android programming portions of the book to make sense.

1 TODO Java Basic Concepts

Java came out sometime in 1995. It's creator is James Gosling who was then working for Sun Microsystems. As such, Java, then, was identified with Sun Microsystems. At the time of this writing, Oracle is the custodian of the Java platform. Java refers to a couple of things. A programming language, a virtual machine, a set of libraries and a set of frameworks.

The programming language allows you to write instructions in a terse, yet English like fashion. It's not as terse as C but not as verbose as say Visual Basic. Java programs are written in program source files which are really nothing more than plain text files. These program source files has an extension of .java. Because the program files are plain text, you can use any plain text editor to create and edit your the source files. As you code more and more, you probably will change your preferences and gravitate to one of more popular program editors e.g. Vim, Emacs, TextMate, SublimeText2 etc. A real program editor has many advantages over plain text editors. As you explore Java programming, the tactical advantages of a program editor will become obvious. Things like line numbers, code folding, syntax coloring, snippet automation etc are some time saving features that quite a lot of coders value. You might too, in time. You might even consider using a full blown IDE like Eclipse, NetBeans or IntelliJ. An Integrated Development Environment can do more than editing of source programs. They can automate setups of debugger, breakpoints, setups of application servers and more.

Although the basic setups of Eclise and NetBeans are covered in this book, all of the examples were coded, built and tested using only Android SDK and program editors — not IDEs.

1.1 TODO Java Compiler

-[] Hello World program, some pictures

Java programs are made up of instructions. The instructions are written a source file and may even look somewhat close to the English language. It should because Java is still considered a high level programming language. But the instructions you write in a source file cannot be understood by the computer. They are not machine readable which means you won't be able to run them straight away. They need to be converted into a form that a machine, more specifically, a Java Virtual Machine, can understand.

Converting a source file to machine code requires a process which is called compilation. It is a process where the high level source code is inspected, pulled apart, analyzed and eventually turned into an unintelligible glob. This glob is not meant to be read by humans. It is meant to be read and executed by a machine. The resulting machine file is called a byte code. The JDK (Java Development Kit) includes a compiler that you can use to convert source files into byte codes. This Java compiler is part of the Java SDK. When you downloaded and installed the JDK, this compiler, together with other Java development tools, were installed into your computer.
